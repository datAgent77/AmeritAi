# Omni / ElevenLabs parity matrix

## Landing parity

| Reference area | Omni implementation target |
| --- | --- |
| Product nav | Dedicated Omni product nav above the marketing stack |
| Hero composition | Large conversational hero with demo shell and dual CTA |
| Proof bar | Customer / capability proof row below hero |
| Feature sequence | Multimodal, workflows, customization, multilingual, integrations, enterprise |
| FAQ and stories | FAQ block and customer-story style grid before final CTA |

## App parity

| Reference area | Omni implementation target |
| --- | --- |
| Left rail | Dense product sidebar focused on build / operate / scale |
| Top bar | Minimal product header with docs, Console bridge, account controls |
| Workspace rhythm | Dense cards, filters, summaries, quick actions |
| Agent detail | Multi-tab agent detail flow with evaluation, tools, knowledge, advanced |
| Monitoring stack | Testing, experiments, versioning, analytics as first-class surfaces |

## Interaction parity

| Reference area | Omni implementation target |
| --- | --- |
| Auth gate | Separate Omni login route that never changes existing `/login` |
| Onboarding path | Landing CTA always routes to `/omni/login?redirect=/omni/app` |
| Workspace switcher | Omni account selector preserved inside the app shell |
| Detail drill-down | Agent detail, channel detail, and Console bridge pages |

## Visual parity

| Reference area | Omni implementation target |
| --- | --- |
| Grid and rhythm | Wide product canvas, compact cards, consistent section spacing |
| Hierarchy | Strong page eyebrow, title, dense supporting summaries |
| Tone | Soft blue / slate gradients with bright surfaces over dark shell |
| Motion cues | Hover elevation and subtle gradient transitions on key tiles |
